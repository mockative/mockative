# Plan: Fix Kotlin/Native `makeValueOf` for Kotlin 2.3.20

## Problem

Kotlin 2.3.20 introduces real downcast checks at reified generic return boundaries on
Kotlin/Native. Mockative's `makeValueOf` on non-JVM platforms returns `Unit as T`, which
now throws `ClassCastException` instead of silently passing.

## Approach

Replace the `Unit as T` hack on Native with real instance creation via KSP-generated code:

1. **Concrete classes** -> `createUninitializedInstance<ConcreteClass>()` (bypasses constructors, like Objenesis on JVM)
2. **Sealed types** -> KSP resolves to a concrete leaf at compile time, then `createUninitializedInstance`
3. **Non-sealed interfaces/abstract classes** -> KSP generates a fake implementation class, then `createUninitializedInstance` on that
4. **Unknown types** -> clear error message

All type resolution happens at compile time because:
- `KClass` on Native lacks reflection (`isSealed`, `isAbstract`, `sealedSubclasses` are JVM-only)
- `createUninitializedInstance<T>()` is reified-only (no KClass overload)

## Scope

- Native only (JS/WasmJS not yet reported as broken)
- KSP processor generates native-specific `makeValueOf` with all known types
- Plugin skips copying `MakeValueOf.kt` for `nativeMain`

## Findings

### Step 0 findings (verified)
- Upgraded Kotlin 2.3.0 -> 2.3.20, KSP 2.3.4 -> 2.3.7, AGP 8.12.3 -> 9.2.0, Gradle 9.2.1 -> 9.5.1
- AGP upgrade required: KSP 2.3.7 + AGP 8.12.3 threw "Kotlin multiplatform Variant API does not support addKspConfigurations() yet"
- AGP 9.2.0 requires Gradle 9.4.1+, so Gradle also upgraded
- `androidLibrary` block deprecated in AGP 9.x, should use `android` instead (warning only)
- **Issue confirmed**: 7 of 45 tests fail with `ClassCastException` on iosSimulatorArm64:
  - `FileServiceTests` (5 failures) ‚Äî uses `any()` with function types / complex types
  - `SealedInterfaceServiceTests` (1 failure) ‚Äî sealed interface valueOf
  - `ValueOfTests.makeValueOfReturnsOtherStuff` (1 failure) ‚Äî direct makeValueOf test

### createUninitializedInstance findings (verified)
- `kotlin.native.internal.createUninitializedInstance<T>()` works on Native
- Requires `@Suppress("INVISIBLE_MEMBER", "INVISIBLE_REFERENCE")` and `@OptIn(InternalForKotlinNative::class)`
- **Reified-only** ‚Äî no KClass overload, so each type must be a compile-time-known reified argument
- This means `makeValueOf` MUST be generated by KSP with explicit `when` branches per type
- Successfully tested with data classes and sealed interface implementations

### KClass limitations on Native
- `isSealed`, `isAbstract`, `isOpen`, `sealedSubclasses` are all JVM-only
- No runtime type introspection possible on Native
- All type classification must happen at KSP compile time

## Steps

### Step 0: Verify the issue ‚úÖ
- [x] Upgrade Kotlin to 2.3.20, KSP to 2.3.7, AGP to 9.2.0, Gradle to 9.5.1
- [x] Run native tests and confirm ClassCastException (7 failures)
- [x] Verify `createUninitializedInstance` works on Native
- [x] Commit version upgrade

### Step 1: KSP processor ‚Äî collect types needing valueOf
- [ ] Scan all function parameter types and return types across `@Mockable` types
- [ ] For each type, classify: concrete class, sealed type (resolve to leaf), interface, abstract class
- [ ] Collect the set of types that will appear in the generated `makeValueOf`

### Step 2: KSP processor ‚Äî generate fakes for interfaces/abstract classes
- [ ] For non-sealed interfaces and abstract classes found in Step 1, generate fake implementation classes
- [ ] Each fake implements/extends the type with all abstract members returning `valueOf()` defaults
- [ ] These fakes are the targets for `createUninitializedInstance` in `makeValueOf`

### Step 3: KSP processor ‚Äî generate native `makeValueOf`
- [ ] Generate `actual fun <T> makeValueOf(type: KClass<*>): T` for native
- [ ] `when` block maps each known type to `createUninitializedInstance<ConcreteOrFakeType>() as T`
- [ ] Sealed types map to `createUninitializedInstance<ResolvedLeaf>()`
- [ ] Interfaces/abstracts map to `createUninitializedInstance<GeneratedFake>()`
- [ ] `else` branch throws `ValueCreationNotSupportedException`

### Step 4: Plugin ‚Äî skip copying native MakeValueOf
- [ ] Modify `MockativeProcessRuntimeTask` or `ResourceManager` to exclude `nativeMain/kotlin/io/mockative/fake/MakeValueOf.kt`
- [ ] The KSP-generated version takes its place

### Step 5: Verify
- [ ] Run `iosSimulatorArm64Test` ‚Äî all 45 tests should pass
- [ ] Run `jvmTest` ‚Äî verify no regressions
- [ ] Commit and update plan
